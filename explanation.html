Proposed Features

I have implemented two additional features: a custom 404 error page for non-existent URLs and a loading animation which is displayed during the POST request. Both improve the user experience.

Custom 404 Error Page

When users enter an invalid or non-existent URL, instead of encountering a generic error, they are presented with a user-friendly 404 page. This provides a better user experience by informing them about the issue and offering a link to return to the API webpage.

Loading Animation

The loading animation indicates that the API is actively processing their request. Since the POST request may take some time to complete, displaying an animation is a useful feature. Furthermore, the submission button becomes temporarily disabled while the animation is visible, preventing any further clicks. This prevents multiple requests from overloading the website and ensures a smoother user experience.
